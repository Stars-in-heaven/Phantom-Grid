What We Set Out
To Do

Started with the goal of building the first version of the Phantom Grid — a dual-panel, offline-first tactical device with three screens, local sensor data, and a file archive terminal.

The device is T-series (Tactical), meaning feature-packed, encrypted, and optimised for security and stability over convenience.

// 02
Key Decisions
1
Series & Version Confirmed
T-V.1 — Tactical series, version 1. Full feature set: all three screens, sensors, GPS, file browser. Firmware hardening planned for Phase 5 after stable firmware is built.
2
Dual ESP32-C6 Architecture
Original plan was a single ESP32-C6. GPIO budget too constrained. Solution: one ESP32-C6 SuperMini per side. Left owns power and sensors. Right owns storage and display.
3
UART via Hinge Link
Both sides mostly independent. When they communicate (e.g. battery status), they use a UART TX/RX link running through the hinge channel over 4 wires.
4
I²C Address Conflict — TCA9548A
Both OLEDs default to 0x3C. Solution: TCA9548A I²C multiplexer (addr 0x70). Each OLED on a separate channel. Forward compatible if more I²C devices added later.
5
SPI Bus Sharing
ILI9341 TFT and microSD share the SPI bus on the right side. 10k pull-up resistor on SD MISO line required to prevent bus contention corrupting TFT data.
6
Strapping Pins Avoided
GPIO8 and GPIO9 are strapping pins on the ESP32-C6 — affect boot behaviour. Deliberately left unconnected on both sides. Right side buttons moved to GPIO21 and GPIO22.
7
Sensors Confirmed for V.1
All three optional sensors included: GPS module (NEO-6M, UART0), temperature sensor (DHT22 or DS18B20), moisture sensor (ADC-capable GPIO). Left side uses both UART peripherals.
// 03
Dual ESP32-C6
Architecture

The original plan was a single ESP32-C6 SuperMini. During Phase 1 planning we identified that a single chip would be too GPIO-constrained to drive everything cleanly — two OLEDs, a TFT, SD card, 5-way nav, tactile buttons, and sensors all at once left no margin.

LEFT
Tactical Overwatch Panel
Owns power and sensors. 0.91" OLED, 1.5" OLED, 5-way nav, GPS, temperature, moisture, battery management.
RIGHT
Archive Command Terminal
Owns storage and display. ILI9341 2.4" TFT, microSD card module, 5x tactile buttons.

Benefits: clean GPIO budget per side, independent development and debugging, split firmware, better security posture — one side compromised does not expose everything.

// 04
Hinge Link

4-wire cable running through the hinge channel carries power and UART communication between both sides.

Wire 1
3.3V
Power from left buck converter to right side
Wire 2
GND
Common ground
Wire 3
TX/RX
UART TX from left (GPIO4) → RX on right (GPIO5)
Wire 4
RX/TX
UART RX to left (GPIO5) ← TX from right (GPIO4)

Use flexible ribbon / FPC cable. Route through hinge channel. Keep wire run as short as physically possible to minimise resistance on 3.3V rail.

// 05
Power Architecture

Left side manages all power. Right side draws from the shared 3.3V rail via hinge.

USB-C → TP4056 (with protection) → LiPo 2000–2500mAh
                                                ↓
                            Buck Converter → 3.3V rail
                                                ↓
                    Left side + Hinge Wire 1 → Right side

Always use TP4056 WITH protection module (DW01 chip on reverse side).

Buck converter must be rated minimum 500mA. Set to 3.3V output before connecting.

Estimated peak draw ~400mA. LiPo 2000–2500mAh gives approx 4–6hrs active use.

TFT backlight is largest power consumer — implement PWM dimming or idle shutoff in firmware.

// 08
Technical Notes
I²C Multiplexer
TCA9548A at address 0x70. Small OLED on channel 0, large OLED on channel 1. Select channel in firmware before addressing each display.
SPI Bus Contention
10k pull-up on SD MISO prevents the SD module from holding the bus low when idle, which would corrupt data being sent to the TFT.
UART Usage — Left Side
Left side uses both UART peripherals: UART0 for GPS (GPIO16/17), UART1 for hinge link to right side (GPIO4/5). No UART left for debug — use JTAG if needed.
